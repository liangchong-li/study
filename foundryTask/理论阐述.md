请简要阐述 Foundry 框架作为智能合约测试 / 部署 / 调试全流程工具链的主要组成部分及其功能。

Foundry的主要组成部分有四个部分
* forge
forge的主要功能是构建、测试、调试、部署和验证智能合约
构建：forge build
测试：forge test
可以执行全部测试，或指定合约、指定测试函数
部署：
可直接通过 forge create 命令部署合约
或使用solidtiy 脚本 ，通过 forge script 命令通过脚本部署
多链部署，通过 .env 配置多环境，然后在命令中选择不同的环境来部署
* cast
cast是一个用于与区块链交互的命令行工具，它可以执行各种操作，如发送交易、查询链上数据、编码/解码数据、计算哈希等。cast设计用于快速、直接地与以太坊节点交互，而无需编写完整的脚本。
* anvil
允许具有分叉功能的本地以太坊开发节点。
它可以从主网上获取一个快照，来部署到本地，利于我们使用真实的数据来调试合约。
* chisel
用于快速原型开发和调试的快速 Solidity REPL。
输入 `chisel` 命令后，会生成一个类似与 `python` 的即时交互窗口，在这个窗口里面不需要创建完整合约框架，可以对简单的代码片段做出即时反馈。
